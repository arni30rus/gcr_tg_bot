# GCR Telegram Bot

Бот для приложения GCR_APP. Бот позволяет клиентам проверять статус своего абонемента, просматривать каталог тренажеров с инструкциями и отправлять обращения администрации.

## Основные функции

- Статус абонемента: Клиент может проверить тип своего абонемента (VIP, Дневной, Безлимит, Льготный), дату окончания и активность.
- Привязка карты клиента: При первом запуске бот запрашивает номер телефона, находит клиента в базе Supabase и привязывает его Telegram-аккаунт.
- Каталог тренажеров: Интерактивное меню с разбивкой по группам мышц. Для каждого тренажера можно прикрепить фото и гифки (GIF) с правильной техникой выполнения.
- Обратная связь: Клиенты могут отправлять текстовые сообщения или фото с подписью напрямую администрации.

## Установка на Ubuntu (Пошаговая инструкция)
1. Подготовка системы

Обновите пакеты и установите необходимые утилиты (Python, pip, venv, git):
```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv git -y
```
2. Клонирование проекта
```bash
git clone https://github.com/arni30rus/gcr_tg_bot.git
cd gcr_tg_bot
```

3. Создание виртуального окружения

Создайте изолированное окружение для Python:
```bash
python3 -m venv venv
```
4. Установка зависимостей

Установите все необходимые библиотеки единой командой из файла requirements.txt:
```bash
venv/bin/pip install -r requirements.txt
``` 
5. Настройка переменных окружения (.env)

Переименуйте файл .env.example в корневой папке проекта в .env и заполните файл своими данными:
```bash
mv .env.example .env
```
6. Настройка каталога тренажеров (equipment.json)

Бот использует локальный файл equipment.json для каталога тренажеров. Заполните его по примеру.

photos и gifs — это массивы с file_id файлов, которые уже были отправлены боту в Telegram.
Получить id файлов можно раскомментировав внизу main.py временный блок. Далее отправляем боту картинку он присылает вам id.
После загрузки всех картинок надо обратно закоментировать блок и перезапустить бота.

## Запуск бота

### Вариант А: Ручной запуск (для тестирования)

Запустите бота напрямую через Python:
```bash
source venv/bin/activate 
python3 main.py
```
Для остановки нажмите Ctrl+C.

### Вариант Б: Запуск как сервис systemd (для постоянной работы)

Чтобы бот работал в фоне, запускался при перезагрузке сервера и сам перезапускался при ошибках, отредактируйте пусть до вашей папки в файле сервиса gcr_tg_bot.service и затем скопируйте его:
```bash
sudo cp gcp_tg_bot.service /etc/systemd/system/gcr_tg_bot.service
```

Перезагрузите конфигурацию systemd, включите автозапуск и запустите сервис:
```bash
sudo systemctl daemon-reload
sudo systemctl enable gcr_tg_bot.service
sudo systemctl start gcr_tg_bot.service
```
